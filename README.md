# OCR Фото Аналізатор

Продвинутий сайт для розпізнавання тексту на фото з використанням нейронної мережі Tesseract.js та передовим препроцесингом зображень.

## 📋 Функціонал

### Основні можливості
- 📸 Перетягування або вибір фото з комп'ютера
- 🧠 Аналіз тексту за допомогою Tesseract OCR
- 💾 Завантаження результату як текстового файлу
- 🌙 Темний/світлий режим з автозбереженням
- 👤 Система користувачів (гість vs VIP через Google)

### Передовий OCR

#### Налаштування препроцесингу
- **Масштабування**: автоматичне поповщення малих зображень до мінімум 500px
- **Адаптивна контрастність**: локальне посилення контрасту по блокам (CLAHE-подібен алгоритм)
- **Оптимальна бінаризація**: алгоритм Otsu для кращого розділення тексту/фону
- **Адаптивна бінаризація Нібласка**: урахування локальної статистики пікселів
- **Морфологічні операції**: видалення дрібних артефактів та шуму

#### Вибір типу документа
- **Документ/Текст** – для стандартних документів та текстів
- **Квитанція/Чек** – оптимізовано для рідкого розташування символів з цифрами
- **Рукопис** – налаштовано для рукописного тексту
- **Низька якість** – для пошкоджених або розпливчастих фото

#### Підтримка мов
- 🇺🇦 Українська + 🇺🇸 Англійська (за замовчуванням)
- 🇺🇦 Тільки українська
- 🇺🇸 Тільки англійська
- 🇷🇺 Російська

#### Додатковий функціонал
- ✅ **Посилене розпізнавання** – включення адаптивної бінаризації та контрасту
- 📊 **Рівень впевненості** – показ процентного рівня впевненості розпізнавання
- 🧹 **Постпроцесинг** – видалення артефактів OCR та нормалізація пробілів

### Система квот
- **Гість**: 5 фото за 5 хвилин (поновлюється автоматично)
- **VIP** (через Google Sign-In): безліміт завантажень

## 🚀 Як запустити локально

Оскільки браузери не дозволяють завантажувати модулі з `file://` протоколу, потрібен локальний веб-сервер.

### ✅ Варіант 1: Python (найпростіший)
```bash
python -m http.server 8000
```
Потім відкрийте http://localhost:8000 у браузері.

### ✅ Варіант 2: Windows batch скрипт
Просто виконайте:
```bash
run-server.bat
```

### ✅ Варіант 3: Windows PowerShell
```bash
.\run-server.ps1
```

### ✅ Варіант 4: Node.js (якщо встановлено)
```bash
npx http-server
```

## 🎯 Алгоритми образу обробки

### Otsu Thresholding
Розраховує оптимальний поріг бінаризації за гістограмою зображення для кращого розділення foreground/background.

### CLAHE (Contrast Limited Adaptive Histogram Equalization)
Розділяє зображення на блоки та застосовує локальне посилення контрасту для кожного блоку окремо.

### Niblack Adaptive Binarization
Розраховує локальне середнє значення та стандартне відхилення для кожного пікселя, адаптуючи поріг до локальних умов.

### Morphological Operations
Операції дилатації та закриття для видалення дрібного шуму та заповнення невеликих прогалин у тексті.

## 💡 Рекомендації для краща розпізнавання

1. **Якість фото** – Використовуйте чітку фотографію з хорошим освітленням
2. **Вибір типу документа** – Виберіть правильний тип для кращого налаштування PSM (Page Segmentation Mode)
3. **Посилене розпізнавання** – Включайте для складних зображень, вимкайте для швидкої обробки
4. **Вибір мови** – Виберіть мови, які присутні у документі, для більшої точності

## 🔒 Безпека

- **Google Sign-In** – Вимагає заміни `YOUR_GOOGLE_CLIENT_ID` на реальний ID перед деплоєм
- **LocalStorage** – Всі дані користувача зберігаються локально у браузері

## 📊 Результати
